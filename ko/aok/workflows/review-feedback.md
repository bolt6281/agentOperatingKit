# 리뷰 피드백 반영 Workflow

> 관리 대상 agent workflow 문서입니다. 사용자 승인 없이 의미를 바꾸는 수정은 하지 않습니다.
> 전제: 이 workflow는 Superpowers 스킬이 설치된 agent 환경을 기준으로 한다.

## 목적

리뷰 피드백을 즉시 기계적으로 반영하지 않고, 타당성과 충돌 여부를 검토한 뒤 작은 단위로 반영한다.

## 절차

1. 리뷰 피드백을 받으면 즉시 수정하지 않고 `receiving-code-review`로 먼저 검토한다.
2. 피드백이 요구사항, 기존 설계, 테스트 결과와 충돌하는지 확인한다.
3. 타당한 피드백은 작은 단위로 반영한다.
4. 동작 변경이 생기면 `test-driven-development` 또는 기존 테스트 보강을 사용한다.
5. 반영 후 `verification-before-completion`으로 관련 검증을 다시 실행한다.

## 완료 전 확인

- 피드백의 타당성을 먼저 검토했는가?
- 반영하지 않은 피드백이 있다면 이유가 명확한가?
- 동작 변경이 생겼다면 테스트나 대체 검증을 보강했는가?
- 관련 검증을 다시 실행했는가?
