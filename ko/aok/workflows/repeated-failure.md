# 반복 실패 / 프로젝트 전용 스킬화 Workflow

> 관리 대상 agent workflow 문서입니다. 사용자 승인 없이 의미를 바꾸는 수정은 하지 않습니다.
> 전제: 이 workflow는 Superpowers 스킬이 설치된 agent 환경을 기준으로 한다.

## 목적

같은 유형의 실수나 실패가 반복될 때, 문서나 스킬을 늘리기 전에 자동화 가능한 해결책을 먼저 찾는다.

## 절차

1. 같은 유형의 실수나 실패가 반복되면 먼저 root cause와 재발 조건을 정리한다.
2. 자동화로 막을 수 있으면 스킬보다 스크립트, 테스트, lint, CI를 우선한다.
3. 판단이 필요한 절차라면 `writing-skills`의 원칙을 참고하되, 프로젝트 전용 스킬은 축약 절차로 작성한다.
4. 프로젝트 전용 스킬은 공식/전역 스킬이 아니라 해당 프로젝트에서만 쓰는 운영 지식으로 둔다.
5. 새 전용 스킬이 필요하면 `.agents/skills/<skill-name>/SKILL.md`와 `.agents/INDEX.md`의 프로젝트 전용 skill 표 갱신안을 먼저 제안하고 사용자 승인 후 생성한다.
6. 프로젝트 전용 skill이 많아지면 사용자 승인 후 `.agents/skills/INDEX.md`를 별도로 만들 수 있다.
7. 이후 같은 유형의 작업을 시작할 때 공통 라우팅에서 `.agents/INDEX.md`의 프로젝트 전용 skill 표를 확인해 해당 스킬을 사용한다.

## 우선순위

1. 테스트, lint, CI로 막을 수 있으면 자동화한다.
2. 자동화가 어렵고 판단이 필요하면 workflow 문서로 남긴다.
3. 반복 빈도가 높고 절차가 안정되면 프로젝트 전용 스킬로 정리한다.

## 완료 전 확인

- 문제의 root cause가 문서 부족인지, 자동화 부족인지 구분했는가?
- 새 문서나 스킬 없이 해결할 수 있는가?
- 새 프로젝트 전용 스킬 생성은 사용자 승인을 받았는가?
