# 조사 / 의사결정 / 기술 선택 Workflow

> 관리 대상 agent workflow 문서입니다. 사용자 승인 없이 의미를 바꾸는 수정은 하지 않습니다.
> 전제: 이 workflow는 Superpowers 스킬이 설치된 agent 환경을 기준으로 한다.

## 목적

구현하지 않고 먼저 질문의 목적과 의사결정 기준을 정리한 뒤, 대안과 리스크를 비교하고 필요한 문서 갱신으로 연결한다.

## 절차

1. 구현하지 않고 먼저 질문의 목적과 의사결정 기준을 정리한다.
2. 필요하면 최소 질문으로 선택지와 평가 기준을 확정한다.
3. 최신성이 중요한 주제이면 공식 문서, primary source, release note, 표준 문서 등 신뢰할 수 있는 출처를 우선 확인한다.
4. 외부 출처를 사용했다면 결정 근거에 출처와 확인 날짜를 남긴다.
5. 대안, 장단점, 리스크, 추천안을 정리한다.
6. 결정이 프로젝트 운영 기준이나 아키텍처에 영향을 주면 관련 문서, `docs/adr.md`, 또는 `docs/adr/INDEX.md`가 가리키는 영역별 ADR을 갱신한다.
7. 결정 이후 구현이 필요해지면 `feature-change.md` 또는 `risky-change.md` workflow로 전환한다.

## 완료 전 확인

- 무엇을 결정해야 하는지 명확한가?
- 평가 기준과 선택지가 명시되어 있는가?
- 최신성이 중요한 주제라면 공식 문서나 primary source를 확인했는가?
- 외부 출처를 사용했다면 출처와 확인 날짜를 남겼는가?
- 추천안이 프로젝트 운영 기준이나 아키텍처에 영향을 주는가?
- 구현으로 전환할 경우 적절한 workflow를 선택했는가?
