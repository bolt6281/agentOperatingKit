# 문서 지도 섹션

이 선택 섹션은 프로젝트에 문서 라우팅, source-of-truth 상태, 검토 상태 관리가 필요할 때만 사용한다. 사용자가 문서 거버넌스를 선택한 뒤 루트 `AGENTS.md`나 프로젝트 문서 색인에 넣는다.

존재하지 않는 문서는 올리지 않는다. 앞으로 만들 수도 있는 후보 문서는 올리지 않는다. 목록에 있는 모든 문서를 기본으로 읽지 않는다.

## 문서 지도

| 문서 | 경로 | 상태 | 기준 여부 | 언제 읽는지 | 언제 갱신하는지 |
|---|---|---|---|---|---|
| `[문서명]` | `[경로]` | active / draft / deprecated / archived | source of truth / 참고 문서 / 충돌 후보 | `[구체적인 읽기 조건]` | `[구체적인 갱신 조건]` |

현재 작업과 관련 있는 문서만 읽는다.

## 읽는 빈도

| 빈도 | 문서 유형 | 규칙 |
|---|---|---|
| 항상 읽음 | 루트 `AGENTS.md` | 매 세션 읽어도 부담 없게 짧게 유지 |
| 라우팅 시 읽음 | `.agents/INDEX.md`, 하위 `AGENTS.md` | 라우팅이나 영역별 규칙이 필요할 때만 읽음 |
| 작업별로 읽음 | 선택된 workflow 파일 | 해당 반복 작업에만 읽음 |
| 참조가 필요할 때 읽음 | ADR, current status, version history | 작업이 해당 기준 문서에 의존할 때만 읽음 |
| 예외 상황에 읽음 | work log, archive | 롤백, 복구, 장애 조사, 회귀 추적, 과거 보존 확인 때만 읽음 |

## 문서 상태

- `active`: 현재 기준 문서
- `draft`: 검토 중인 초안
- `deprecated`: 더 이상 기준으로 쓰지 않지만 전환 참고용으로 남김
- `archived`: 과거 기록 보존용

`deprecated`와 `archived` 문서는 새 작업의 기준으로 사용하지 않는다.

## 유지 규칙

다음 변경이 발생하면 같은 작업 안에서 문서 지도도 갱신한다.

- 기준 문서를 생성, 이동, 이름 변경, 삭제할 때
- 어떤 문서가 더 이상 source of truth가 아니게 될 때
- 문서를 `docs/archive/`로 이동할 때
- 제품 정책, API 계약, 배포 절차, 데이터 구조, 운영 기준이 바뀔 때
- 문서 간 충돌을 해결해 하나의 문서를 우선 기준으로 정했을 때

`docs/work-log.md`는 일반 작업 시작 시 읽지 않는다. 평소에는 기록 대상으로만 사용하고, 롤백, 복구, 장애 조사, 회귀 원인 추적 때만 읽는다.
