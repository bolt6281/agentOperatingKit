# 현재 상태 문서 템플릿

이 템플릿은 장기 운영 프로젝트에서 지금 기준으로 믿어야 하는 운영 상태를 짧게 기록할 때 사용한다.

`docs/current-status.md`는 작업 일지가 아니다. 과거 히스토리는 `docs/VERSION_HISTORY.md`, 장기 메모는 필요한 경우 `docs/work-log.md`, 결정 근거는 ADR에 둔다.

# 현재 상태

> 마지막 갱신: `[YYYY-MM-DD]`

## 현재 기준

- 제품/서비스 상태: `[운영 중 / 개발 중 / 보류 / 기타]`
- 현재 주요 버전 또는 배포 기준: `[버전, 브랜치, 태그, 배포 환경]`
- source of truth 문서: `[링크]`

## 운영 상태

- 배포 환경: `[환경과 접근 경로]`

- 최근 확인된 정상 동작: `[날짜와 확인 방법]`

## 남은 위험과 확인 필요 항목

| 항목 | 상태 | 다음 행동 |
|---|---|---|
| `[위험 또는 불확실성]` | 확인 필요 | `[확인 방법]` |

## 갱신 규칙

- 배포 기준, 검증 명령, 외부 의존성, 운영 상태가 바뀌면 갱신한다.
- 단일 작업의 상세 진행 내역은 이 문서에 누적하지 않는다.
- 실제 프로젝트에 적용할 때는 placeholder를 남기지 않는다.
