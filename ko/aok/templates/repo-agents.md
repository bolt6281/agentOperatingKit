# 프로젝트 AGENTS.md

이 문서는 최소 루트 `AGENTS.md` 템플릿이다. 매 세션 읽어도 부담 없는 크기로 유지한다. 작은 프로젝트는 50-100줄 안팎을 우선한다.

선택적 workflow, 긴 문서 거버넌스 규칙, 작업 로그, ADR 내용, archive 요약을 이 파일에 직접 붙이지 않는다. 선택적 세부사항은 명시적 읽기 조건 뒤에 둔다.

## 프로젝트

- 목적: `[무엇을 하는 제품/도구]`
- 주요 사용자: `[사용자 유형 또는 사용 맥락]`
- 현재 우선순위: `[안정성/속도/감사 가능성/UX/운영 편의성 등]`

## 작업 전

- 이 파일을 먼저 읽는다.
- 아래 목록 중 현재 작업에 필요한 문서만 읽는다.
- 사용자 승인 없이 프로젝트 문서를 생성, 이동, archive, deprecated 처리, 삭제하지 않는다.
- 요구사항, 문서, 검증 명령이 충돌하면 파일을 바꾸기 전에 충돌을 보고한다.

## 검증 라우팅

| 변경 | 먼저 읽을 문서 | 검증 방법 |
|---|---|---|
| 코드 | `[경로 또는 "없음"]` | `[test/lint/typecheck/build 명령]` |
| 문서 | `[경로 또는 "없음"]` | `[링크 검사 또는 영향받는 참조 확인]` |
| 위험 변경 | `[경로 또는 승인 규칙]` | `[rollback/CI/deploy 검증]` |

여러 영역을 함께 바꾸면 관련 검증 라우팅을 모두 적용한다.

## 읽을 문서

현재 프로젝트에서 실제로 사용하는 기존 문서만 적는다. 읽는 조건은 구체적으로 적는다.

| 상황 | 읽을 문서 |
|---|---|
| 기능 작업 | `[경로 또는 없음]` |
| 버그 수정 | `[경로 또는 없음]` |
| 아키텍처 또는 제품 결정 | `[경로 또는 없음]` |
| 배포, CI, 인증, DB 등 위험 작업 | `[경로 또는 없음]` |
| 롤백, 복구, 장애 조사, 회귀 추적 | `[경로 또는 없음]` |

현재 작업에 필요한 문서만 읽는다.

## 선택 확장

- 문서의 source-of-truth 상태, 명시적 읽기 조건, 검토 상태가 필요할 때만 [document-map.md](document-map.md)를 추가한다.
- agent 지침 파일에 승인 규칙이 필요할 때만 [agent-instruction-governance.md](agent-instruction-governance.md)를 추가한다.
- workflow, 하위 `AGENTS.md`, 프로젝트 전용 skill 라우팅이 필요할 때만 `.agents/INDEX.md`를 추가한다.
- Superpowers를 사용하고 반복 작업 절차가 안정된 프로젝트에만 `.agents/workflows/*.md`를 추가한다.
