# Agent 지침 변경 제안 양식

이 양식은 파일로 저장하기보다, agent가 채팅에서 사용자 승인을 요청할 때 복사해 쓰는 승인 양식이다.

프로젝트에 적용된 뒤에는 보통 `.agents/INDEX.md`나 루트 `AGENTS.md`에서 이 양식을 참조한다. 별도 파일로 생성할 필요는 없다.

```md
## Agent 지침 변경 제안

- 변경 유형: 새 하위 `AGENTS.md` 생성 / 기존 지침 의미 변경 / workflow 문서 의미 변경
- 대상 경로:
- 적용 범위:
- 필요한 이유:
- 제안 규칙:
  - ...
- 기존 규칙과 충돌 여부:
- 검증 방법:
- 사용자 승인 필요: 예
```

예시:

```md
## Agent 지침 변경 제안

- 변경 유형: 새 하위 `AGENTS.md` 생성
- 대상 경로: `apps/web/AGENTS.md`
- 적용 범위: `apps/web/**`
- 필요한 이유: 프론트엔드 검증 명령과 컴포넌트 작성 규칙이 루트 규칙보다 구체적이다.
- 제안 규칙:
  - UI 변경 후 `pnpm --filter web test`를 실행한다.
  - 새 컴포넌트는 먼저 `packages/ui`를 확인한다.
- 기존 규칙과 충돌 여부: 없음
- 검증 방법: `pnpm --filter web lint`, `pnpm --filter web test`
- 사용자 승인 필요: 예
```
