# Agent 지침 거버넌스 섹션

이 선택 섹션은 agent 지침 파일을 관리 대상 정책 파일로 두고 변경 승인 규칙이 필요할 때만 사용한다. 짧은 루트 `AGENTS.md`만 필요한 작은 프로젝트에는 추가하지 않는다.

## 관리 대상 지침 파일

- `AGENTS.md`
- `**/AGENTS.md`
- `.agents/**`

## 승인 없이 가능한 변경

다음 변경은 기존 의미를 보존하는 경우에만 승인 없이 가능하다.

- 오탈자 수정
- 깨진 내부 링크 수정
- 실제 파일 이동에 따른 경로 정정
- 명백히 오래된 날짜나 파일명 정정
- 표 형식 정렬, Markdown 포맷 정리
- `.agents/INDEX.md`에 이미 승인된 파일의 누락된 메타데이터 보완
- 삭제된 파일을 `.agents/INDEX.md`에서 `deprecated` 또는 `archived`로 표시

의미가 바뀔 가능성이 있으면 승인 필요 변경으로 다룬다.

## 사용자 승인이 필요한 변경

- 새 루트 또는 하위 `AGENTS.md` 생성
- 새 `.agents/workflows/*.md` 생성
- agent의 필수/금지 행동 변경
- 검증 명령 추가, 제거, 변경
- 적용 범위, 소유자, 책임 경계 변경
- workflow 단계 추가, 제거, 순서 변경
- 문서 생성, 승인, 위험 작업, archive 정책 변경

## 명시적 사용자 요청

사용자가 직접 지침 수정, workflow 추가, 구조 생성을 요청한 경우에는 해당 요청 범위 안에서만 수정한다.

요청 범위를 넘는 추가 지침이나 workflow는 먼저 제안하지 않고 만들지 않는다.

## 애매한 경우

변경이 단순 정비인지 정책 변경인지 애매하면 정책 변경으로 간주하고 먼저 제안한다.

승인 후에는 해당 파일을 생성하거나 의미를 바꾸는 수정을 하고, 필요한 경우 같은 작업 안에서 `.agents/INDEX.md`도 갱신한다.
